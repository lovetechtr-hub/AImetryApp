# Руководство по разработке

## Архитектура

Проект использует следующую архитектуру:

### Многоуровневая архитектура

1. **UI Layer** (`ui/`)
   - Compose Multiplatform экраны
   - Компоненты UI
   - Навигация

2. **Domain Layer** (`domain/`)
   - Модели данных
   - Use cases (будут добавлены при необходимости)

3. **Data Layer** (`data/`)
   - Репозитории
   - Локальное хранилище
   - API клиенты

4. **API Layer** (`api/`)
   - HTTP клиенты
   - Endpoints
   - Модели API

## Структура кода

### Модели данных

Все модели данных находятся в:
- `api/models/` - модели для API ответов
- `domain/model/` - доменные модели

### Репозитории

Репозитории находятся в `data/repository/` и содержат бизнес-логику работы с данными.

### UI компоненты

- `ui/screens/` - экраны приложения
- `ui/components/` - переиспользуемые компоненты
- `ui/theme/` - тема и стили

## Стиль кода

- Используйте Kotlin coding conventions
- Используйте Compose best practices
- Следуйте принципам Clean Architecture
- Документируйте публичные API

## Тестирование

- Unit тесты должны покрывать бизнес-логику
- UI тесты для критических экранов
- Integration тесты для API

## Коммиты

Используйте conventional commits:
- `feat: добавлена новая функция`
- `fix: исправлена ошибка`
- `refactor: рефакторинг кода`
- `docs: обновлена документация`
